# UiPath Process App Configuration

This document outlines the UI configuration for the SentinelFin Process App that compliance officers will use to interact with the Maestro case at the human gates.

## Gate 1: Triage Analyst Review
**Trigger**: Agents 1-3 have completed.
**Target SLA**: Under 4 hours from alert creation.

### UI Fields:
- **Case Summary Box**:
  - `Alert Type`: (e.g., SUSPECTED_STRUCTURING)
  - `Risk Score`: (Calculated by Agent 3)
  - `Account History Link`: (Provided by Agent 1)
  - `Sanctions Status`: (Provided by Agent 2)
- **Action Buttons**:
  - `[Escalate to SAR]` -> Proceeds to Stage 2 (Deep Investigation).
  - `[Close Alert (False Positive)]` -> Requires mandatory justification text box. Ends case.
  - `[Enhanced Due Diligence (EDD)]` -> Routes back to Agent 1 for a wider data pull.

## Gate 2: BSA Officer Signature (Blocking Gate)
**Trigger**: Agents 6 and 7 have drafted the narrative and populated the form.
**Target SLA**: Before Day 28.

### UI Fields:
- **SAR Narrative Review Tab**: Read-only text box with the generated narrative.
- **Form 111 Review Tab**: Data grid showing populated fields.
- **Signature Panel**:
  - `Digital Signature Checkbox`: "I certify that I have reviewed this SAR..."
  - `Approval Button`: `[Sign and Approve for E-Filing]` -> Triggers Agent 8.
  - `Rejection Button`: `[Send back for edits]` -> Opens text box for feedback, loops back to Agent 6.
*Note: The Maestro case MUST enforce that the 'Digital Signature Checkbox' is True before Agent 8 can execute.*

## Gate 3: Ongoing Monitoring
**Trigger**: 90 days after initial submission.

### UI Fields:
- **Previous SAR Link**: Link to original submission.
- **New Activity Summary**: (Generated by a scheduled Agent 1 run).
- **Decision Buttons**:
  - `[File Continuing SAR]`
  - `[Close Case (No new suspicious activity)]`
